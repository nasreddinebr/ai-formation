# Project Goal: AI Curriculum Website Development

I have an AI training curriculum document, `formation-ia-complete.md`, titled "From Zero to AI Engineer — A complete, progressive, and career-oriented curriculum." Your task is to transform this content into a modern, multi-page website based on the specifications below.

## 1. Technology & Standards

* **Tech Stack Selection:** Suggest the most suitable, modern technology stack (e.g., Next.js with Tailwind CSS). Justify your choice based on performance, scalability, and maintainability.
* **Clean Code & Maintainability:** The generated code must be highly maintainable, modular, and strictly adhere to **Clean Code principles** (e.g., meaningful naming, small focused functions, DRY principles, and clear folder architecture).
* **Best Practices:** Use stable, long-term support (LTS) versions for all dependencies. Implement accessible UI and responsive design best practices.
* **Local Setup:** Provide a clear, step-by-step guide on how to set up the project locally for testing. Explicitly list all necessary installations (Node.js, package managers, CLI tools) and environment setup commands.

## 2. Architecture & Features

* **Homepage:** Display the curriculum structure with clear navigation to chapters.

* **Authentication (Structure Only):** Create "Sign-in" and "Sign-up" page skeletons to lay the groundwork for future user-specific features.
* **Bookmarking:** Implement a client-side feature using LocalStorage so the homepage suggests a "Resume where you left off" link based on the user's last visited chapter.

## 3. Design & UX

* **Theme:** Modern, clean, and UX-focused interface.
* **Color Palette:** Use: Primary (`#355872`), Secondary (`#7AAACE`), Accent (`#9CD5FF`), and Neutral (`#F7F8F0`).
* **Features:** Include a Dark Mode/Light Mode toggle.
* **Media:** Use open-source, royalty-free images via external URLs.

## 4. Execution

1. **Propose:** Suggest the tech stack, justify it, and provide the local setup guide.
2. **Architect:** Define the file structure to ensure high maintainability.
3. **Develop:** Build the UI components and integrate the content, prioritizing readable and well-commented code.
4. **Verify:** Ensure all features, including the bookmarking logic, follow the project constraints.
